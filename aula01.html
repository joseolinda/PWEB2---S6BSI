O que é JavaScript?

JavaScript (JS) é uma linguagem de programação de alto nível, interpretada e multi-paradigma (suporta programação procedural, orientada a objetos e funcional). Em JS (quase) tudo é um objeto, incluindo funções.

É uma das três principais tecnologias da World Wide Web, junto com HTML e CSS.

  • HTML: Define a estrutura e o conteúdo das páginas web.
  • CSS: Define o estilo e a apresentação visual.
  • JavaScript: Adiciona interatividade e comportamento dinâmico.

Curiosidade: JavaScript não tem relação direta com a linguagem Java, apesar da similaridade no nome. A escolha do nome foi uma decisão de marketing na época.

4/30

Uma Breve História

  • Criado em 1995 por Brendan Eich na Netscape Communications.
  • Desenvolvido em apenas 10 dias!
  • Originalmente chamado de Mocha, depois LiveScript, e finalmente JavaScript.
  • Padronizado em 1997 pela ECMA International como ECMAScript, para garantir a interoperabilidade entre navegadores.
5/30

Propósito e Aplicações

Inicialmente, o propósito era adicionar interatividade a páginas web (validação de formulários, animações, etc.). Hoje, suas aplicações são vastas:

  • Front-end Web: Manipulação do DOM, interatividade com o usuário (Frameworks como React, Angular, Vue.js).
  • Back-end: Servidores web com Node.js e frameworks como Express.js e Next.js.
  • Mobile Apps: Usando frameworks como React Native e Ionic.
  • Desktop Apps: Com Electron (ex: VS Code, Slack).
  • Jogos: Bibliotecas como Phaser e Three.js.
  • Internet das Coisas (IoT).

Estrutura do Código JavaScript

Um programa em JavaScript é uma sequência de declarações (statements). Declarações são como frases em uma linguagem e são geralmente terminadas com ponto e vírgula (;).

O ponto e vírgula é opcional em muitos casos devido à "Inserção Automática de Ponto e Vírgula" (ASI), mas é uma boa prática sempre utilizá-lo para evitar ambiguidades.

let numero = 10;

console.log(numero);

if (numero > 5) {
    console.log("O número é maior que 5.");
}
7/30

Modo Estrito: "use strict"

O "modo estrito" é uma forma de optar por uma variante restrita do JavaScript. Ele elimina alguns "erros silenciosos" do JavaScript, fazendo com que eles lancem exceções.

Para ativá-lo, adicione "use strict"; no início do seu script ou de uma função.

"use strict";

variavelNaoDeclarada = 10; // ReferenceError

let x = 3.14;
delete x; // SyntaxError
Recomendação: Sempre use o modo estrito. Ele ajuda a escrever um código mais seguro e robusto.
8/30

Variáveis

Variáveis são "contêineres" nomeados para armazenar valores. Em JavaScript, podemos declarar variáveis usando var, let, e const.

let mensagem = "Olá, mundo!";
let idade = 25;
const PI = 3.14159;

Os nomes de variáveis (identificadores) podem conter letras, dígitos, underscores ( _ ) e cifrões ( $ ), mas não podem começar com um dígito.

9/30

var, let e const

Introduzidos no ES6 (ECMAScript 2015), let e const oferecem um escopo de bloco e são as formas RECOMENDADA de declarar variáveis hoje.

  • var:
    • Escopo de função.
    • Pode ser redeclarada e reatribuída.
    • Sofre hoisting (içamento) completo.
  • let:
    • Escopo de bloco {...}.
    • Pode ser reatribuída, mas não redeclarada no mesmo escopo.
    • Sofre hoisting, mas fica na "Temporal Dead Zone" (TDZ) até a declaração.
  • const:
    • Escopo de bloco.
    • Não pode ser reatribuída. Deve ser inicializada na declaração.
    • A "constância" se refere à referência; se for um objeto, suas propriedades podem ser alteradas.

Dica: Use const por padrão. Use let apenas quando souber que a variável precisará ser reatribuída. Evite var para evitar confusões com escopo.

10/30

Escopo de Bloco vs. Escopo de Função

function testeEscopo() {
    var x = 10;
    if (true) {
        var x = 20; // Mesma variável x
        let y = 30;
        console.log("Dentro do if (var x):", x); // 20
        console.log("Dentro do if (let y):", y); // 30
    }
    console.log("Fora do if (var x):", x);   // 20
    console.log("Fora do if (let y):", y); // ReferenceError: y is not defined
}

testeEscopo();

let e const respeitam o bloco if, enquanto var "vaza" para o escopo da função inteira.

Tipos de Dados

JavaScript é uma linguagem de tipagem dinâmica. Isso significa que você não precisa declarar o tipo de uma variável; ele é determinado automaticamente no momento da atribuição.

Os tipos são divididos em Primitivos e Compostos.

Sugestão de leitura: GeeksforGeeks - JavaScript Data Types

12/30

Tipos Primitivos

Valores primitivos são imutáveis (não podem ser alterados).

  • Number: Para números inteiros ou de ponto flutuante. Ex: 42, 3.14. Inclui valores especiais como Infinity, -Infinity e NaN (Not-a-Number).
  • String: Sequências de caracteres, delimitadas por aspas simples ('), duplas (") ou crases (` - template literals). Ex: "Olá".
  • Boolean: Representa valores lógicos. Apenas dois valores: true e false.
  • Undefined: Uma variável que foi declarada mas não inicializada. Valor padrão.
  • Null: Representa a ausência intencional de um valor de objeto.
  • Symbol (ES6): Um tipo de dado cujas instâncias são únicas e imutáveis.
  • BigInt (ES2020): Para números inteiros de precisão arbitrária.
13/30

Verificação de tipos com typeof

O operador typeof retorna uma string indicando o tipo de um operando.

console.log(typeof 42);          // "number"
console.log(typeof "JavaScript"); // "string"
console.log(typeof true);        // "boolean"
let semValor;
console.log(typeof semValor);    // "undefined"
console.log(typeof null);        // "object" (um bug histórico da linguagem! Só aceita!)
console.log(typeof Symbol('id'));// "symbol"
console.log(typeof 123n);        // "bigint"
console.log(typeof NaN);         // "number"

JS e suas bizarrices peculiaridades: NaN significa "Not-a-Number", mas seu tipo é number. Existe explicação para a maioria dessas peculiaridades, leia a documentação oficial para entender melhor: https://ecma-international.org/publications-and-standards/standards/ecma-262/.

14/30

Tipo objeto (Object)

O tipo Object é uma coleção de propriedades, onde cada propriedade é uma associação entre uma chave (ou nome) e um valor.

Quase tudo em JavaScript que não é um tipo primitivo é um objeto.

  • Objeto Literal: { chave: 'valor' }
  • Array: Um tipo especial de objeto, usado para listas ordenadas. [1, "dois", true]
  • Function: Funções também são objetos em JavaScript, o que permite tratá-las como valores.
let pessoa = {
    nome: "Ana",
    idade: 30,
    ativa: true
};

let cores = ["vermelho", "verde", "azul"];

function saudar() {
    console.log("Olá!");
}

console.log(typeof pessoa); // "object"
console.log(typeof cores);  // "object" (Array.isArray(cores) é true)
console.log(typeof saudar); // "function"
15/30

Conversão de tipos (type coercion)

JavaScript frequentemente converte valores de um tipo para outro automaticamente. Isso é chamado de conversão implícita ou coerção.

Também podemos converter tipos de forma explícita.

16/30

Conversão Implícita (Coerção)

Ocorre quando operadores são aplicados a valores de tipos diferentes.

// O operador '+' com uma string força a concatenação
console.log("5" + 1);    // "51" (número 1 vira string '1')
console.log(1 + "5");    // "15"

// Operadores matemáticos (-, *, /) tentam converter para número
console.log("10" - 5);   // 5
console.log("10" * 2);   // 20
console.log("dez" * 2);  // NaN (Not-a-Number)

// O operador de igualdade abstrata (==) faz coerção
console.log(5 == "5");   // true
console.log(0 == false); // true
console.log(null == undefined); // true

// Para evitar coerção na comparação, use igualdade estrita (===)
console.log(5 === "5");   // false
console.log(0 === false); // false
17/30

Conversão Explícita

Quando queremos controlar a conversão, usamos funções built-in.

let valorString = "123.45";

// Para Número
let valorNumero1 = Number(valorString);      // 123.45
let valorNumero2 = parseInt(valorString);    // 123 (ignora parte decimal)
let valorNumero3 = parseFloat(valorString);  // 123.45

console.log(typeof valorNumero1); // "number"

let valorNumero = 99;

// Para String
let valorString1 = String(valorNumero);      // "99"
let valorString2 = valorNumero.toString();   // "99"

console.log(typeof valorString1); // "string"

// Para Booleano
console.log(Boolean(0));        // false
console.log(Boolean(""));       // false
console.log(Boolean(null));     // false
console.log(Boolean("olá"));    // true
console.log(Boolean(100));      // true
18/30

Hoisting (Içamento)

Hoisting é um comportamento do JavaScript onde as declarações de variáveis (com var) e funções são movidas para o topo de seu escopo antes da execução do código.

Apenas a declaração é içada, não a inicialização.
19/30

Hoisting com var

Com var, a declaração é movida para o topo e a variável é inicializada com undefined. 

// Como escrevemos o código:
console.log(nome); // undefined
var nome = "Maria";
console.log(nome); // "Maria"

// Como o JavaScript interpreta (após o hoisting):
var nome;
console.log(nome); // undefined
nome = "Maria";
console.log(nome); // "Maria"
20/30

Hoisting com let e const

let e const também são "içadas", mas elas não são inicializadas. Acessá-las antes da declaração resulta em um ReferenceError. A área entre o início do bloco e a declaração é chamada de Temporal Dead Zone (TDZ).

// console.log(idade); // ReferenceError: Cannot access 'idade' before initialization
let idade = 30;
console.log(idade); // 30

// O mesmo acontece com const
// console.log(cidade); // ReferenceError
const cidade = "São Paulo";
console.log(cidade);
Este comportamento nos força a declarar as variáveis antes de usá-las, o que leva a um código mais limpo e menos propenso a erros.

Interações com o Navegador

JavaScript no navegador fornece funções para interagir com o usuário de forma simples e direta.

Vamos ver quatro interações básicas:

  • alert(): Exibe uma mensagem.
  • prompt(): Pede ao usuário para inserir um texto.
  • confirm(): Pede uma confirmação (OK/Cancelar).
  • console.log(): Escreve no console do desenvolvedor.
22/30

alert()

Mostra uma caixa de diálogo modal com uma mensagem e um botão OK. Pausa a execução do script até que o usuário clique em OK.

alert("Bem-vindo ao nosso site!");

console.log("Esta linha só executa depois que o alert é fechado.");
Use com moderação. O uso excessivo de alert pode ser irritante para o usuário.
23/30

prompt()

Exibe uma caixa de diálogo com uma mensagem, um campo de texto e botões OK/Cancelar.

Retorna o texto digitado pelo usuário ou null se o usuário clicar em Cancelar ou fechar a caixa.

let nome = prompt("Qual é o seu nome?", "Visitante");

if (nome) { // Verifica se o usuário não clicou em Cancelar
    alert("Olá, " + nome + "!");
} else {
    alert("Você não quis dizer seu nome.");
}

O segundo argumento da função prompt() é opcional e serve como valor padrão para o campo de texto.

24/30

confirm()

Exibe uma caixa de diálogo com uma pergunta e botões OK/Cancelar.

Retorna true se o usuário clicar em OK e false caso contrário.

let querSair = confirm("Você tem certeza que deseja sair desta página?");
if (querSair) {
    console.log("Usuário confirmou a saída.");
} else {
    console.log("Usuário cancelou a ação.");
}
25/30

console.log()

É a ferramenta mais importante para depuração (debugging). Ela exibe dados no console de ferramentas do desenvolvedor do navegador (geralmente aberto com F12 ou Ctrl+Shift+I).

Diferente das outras, não interrompe a execução do código nem interfere na interface do usuário.

let a = 10;
let b = 20;
let soma = a + b;

console.log("O valor de 'a' é:", a);
console.log("O valor de 'b' é:", b);
console.log("O resultado da soma é:", soma);

let usuario = { nome: "Carlos", id: 123 };
console.log("Dados do usuário:", usuario);
26/30

Console do navegador

O console é uma ferramenta poderosa. Além de console.log(), existem outros métodos úteis:

  • console.error(): Exibe uma mensagem de erro.
  • console.warn(): Exibe uma mensagem de aviso.
  • console.table(): Exibe dados tabulares (objetos, arrays) de forma organizada.
27/30

Exemplo prático

"use strict";

alert("Olá! Vamos coletar algumas informações.");

const anoNascimento = prompt("Em que ano você nasceu?");
const anoAtual = 2025; // Podemos obter isso dinamicamente depois

if (anoNascimento) {
    // Conversão explícita de string para número
    const idade = anoAtual - parseInt(anoNascimento);
    console.log("Cálculo da idade:", anoAtual, "-", anoNascimento, "=", idade);


    alert("Você tem (ou fará) " + idade + " anos em " + anoAtual + ".");

    const confirmaIdade = confirm("A informação está correta?");

    if (confirmaIdade) {
        console.log("O usuário confirmou a idade de " + idade + " anos.");
    } else {
        console.warn("O usuário não confirmou a idade.");
    }


} else {
    console.error("O usuário não informou o ano de nascimento.");
    alert("Operação cancelada.");
}

Revisão: valores, tipos e operadores

  • Tudo em JavaScript é um "valor" com um "tipo" associado.
  • Conhecemos 7 tipos primitivos (Number, String, Boolean, etc.) e o tipo composto Object.
  • JavaScript é de tipagem dinâmica.
  • A conversão de tipos (coerção) acontece com frequência. Use === para comparações estritas e evite surpresas.

Revisão: estrutura do programa

  • Um programa é uma série de declarações (statements). Use ;.
  • Sempre inicie seus scripts com "use strict"; para um código mais seguro.
  • Use let e const para declarar variáveis. Elas têm escopo de bloco e evitam problemas comuns do var.
  • Hoisting é o mecanismo que move declarações para o topo, mas let/const introduzem a TDZ, forçando melhores práticas.
  • console.log() é seu melhor amigo para entender e depurar o código.
